# AI Memory System Design

This document explains how the memory system of the AI **لين** operates.

The system stores structured information instead of full conversations.

---

# Core Concept

Instead of storing chat logs, the AI extracts structured memory entries.

Example:

User message  
أنا أحب القهوة

Stored memory:

type: preference  
user: 123456  
memory: يحب القهوة

This keeps the memory database clean and relevant.

Structured memories allow the AI to recall meaningful information without storing entire conversations.

---

# Memory Types

User Memory  
Information about specific users.

Server Memory  
Observations about server activity.

Conversation Memory  
Short-term context used during conversation.

Developer Memory  
Information related to project development.

---

# Memory Pipeline

Message  
→ memory extraction  
→ memory classification  
→ memory validation  
→ duplicate detection  
→ memory storage  
→ memory retrieval

This pipeline allows the AI to convert natural conversation into structured long-term knowledge.

---

# Implementation

Long-term memory:

systems/aiMemorySystem.js  
repositories/memoryRepository.js

Short-term conversation memory:

utils/memoryManager.js

Long-term memory is stored in the database, while short-term memory is stored temporarily in runtime memory.

---

# Memory Extraction

The AI extracts memories automatically from user messages.

Examples:

"اسمي سعود"  
→ name memory

"احب القهوة"  
→ preference memory

"انا مهتم بالبرمجة"  
→ interest memory

"اعتقد ان الذكاء الاصطناعي مهم"  
→ opinion memory

These memories are classified and stored for future context.

---

# Memory Ranking System

Not all memories are equally important.

The system assigns importance scores based on:

* personal facts (name, age)
* preferences
* interests
* opinions
* message length

More important memories receive higher relevance scores.

---

# Memory Similarity Detection

Before storing a memory the system checks for duplicates.

Duplicate detection works by:

* extracting keywords
* comparing similarity between memories
* rejecting near-duplicate memories

This prevents memory spam and keeps the database clean.

---

# Memory Decay

Older memories become slightly less relevant over time.

The system calculates a decay factor based on:

memory age  
memory importance  
message relevance

This ensures that newer and more relevant memories are prioritized.

---

# Memory Search

When a user sends a message the AI searches for relevant memories.

Process:

User Message  
→ keyword extraction  
→ similarity scoring  
→ ranking memories  
→ selecting top results  
→ injecting memories into AI prompt

Only the most relevant memories are injected into the conversation.

Maximum injected memories:

5 relevant memories per message.

---

# Server Observation Memory

The AI also generates server-level memories automatically.

These memories are created by observation systems.

Examples:

* active server members
* frequently discussed topics
* trending keywords
* community activity patterns

Generated by:

aiObservationSystem  
aiSocialGraphSystem

---

# Social Relationship Memory

The system can detect relationships between members.

Example memory:

"Saud interacts frequently with Ahmed"

This information is generated by:

aiSocialGraphSystem

These memories allow the AI to understand server social dynamics.

---

# Memory Safety Filters

The system prevents sensitive information from being stored.

Blocked content includes:

* passwords
* tokens
* credit card numbers
* banking data
* secrets

Any message containing sensitive keywords is rejected from storage.

---

# Memory Limits

User memory limit: **50 memories**

Server memory limit: **20 memories**

Relevant memory injection limit: **5 memories**

When limits are exceeded the system automatically removes the oldest memories.

This prevents uncontrolled database growth.

---

# Memory Capabilities

The system supports:

* automatic memory extraction
* duplicate detection
* relevance ranking
* contextual memory injection
* sensitive data filtering
* server observation memories
* social relationship detection

This enables the AI **لين** to maintain long-term awareness of users and servers while keeping memory structured and efficient.

11- SYSTEM_FLOW.md









# System Flow

This document explains the runtime execution flow of the system.

The Bot Core processes Discord events through multiple layered systems that provide AI intelligence, gameplay features, and monitoring.

---

# Message Processing

When a message is received in a Discord server the following pipeline is executed.

Discord Message  
→ events/messageCreate.js  
→ guildManager.getGuild  
→ aiObservationSystem.observeMessage  
→ aiSocialGraphSystem.detectRelationships  
→ aiSystem.ensureAIEnabled  
→ aiAutoReplySystem  
→ aiBrainSystem.detectIntent

At this stage the system decides whether the message should trigger an AI response or a command-like intent.

---

# AI Response Flow

User Message  
→ aiAutoReplySystem  
→ aiRateLimitSystem  
→ aiBrainSystem.detectIntent

If an intent is detected:

→ aiBrainSystem.handleIntent  
→ system response generated  
→ message sent to Discord

If no intent is detected:

→ aiTokenUsageSystem  
→ aiHandler  
→ aiMemorySystem.injectMemoriesIntoContext  
→ aiKnowledgeSystem.injectKnowledge  
→ OpenAI API  
→ aiResponseFormatterSystem  
→ response stored in conversation memory  
→ message sent to Discord

This flow allows the AI to combine memory, knowledge, and context before generating responses.

---

# XP / Level Flow

When a message is posted, the leveling system may also process the message.

Discord Message  
→ xpSystem.ensureXPEnabled  
→ levelSystem.addXP (with internal cooldown protection)  
→ level calculation  
→ level up event  
→ level up message sent

This enables progression systems within the server.

---

# Memory Flow

The AI memory system stores structured user information extracted from conversations.

Message  
→ aiMemorySystem.extractMemoryFromMessage  
→ memory classification  
→ duplicate detection  
→ memory storage (memoryRepository)

During conversations:

User Message  
→ aiMemorySystem.searchRelevantMemories  
→ relevant memories injected into prompt

This enables long-term contextual awareness.

---

# Knowledge Flow

The knowledge system allows the AI to learn structured information from conversations.

Message  
→ aiKnowledgeSystem.learnFromMessage  
→ embedding generation (OpenAI embeddings)  
→ knowledge storage (knowledgeRepository)

During AI responses:

User Message  
→ embedding generation  
→ semantic vector search  
→ relevant knowledge injected into AI context  
→ AI response generation

---

# Observation Flow

The observation system monitors server activity to generate contextual memories.

Message  
→ aiObservationSystem.observeMessage  
→ user activity tracking  
→ topic keyword extraction  
→ trending topic detection  
→ server memory generation

---

# Social Graph Flow

The social graph system detects relationships between members.

Message with user mentions  
→ aiSocialGraphSystem.detectRelationships  
→ interaction tracking  
→ relationship threshold detection  
→ relationship memory stored

This enables the AI to understand social dynamics inside the server.

---

# System Interaction Summary

The Bot Core processes each message through multiple subsystems simultaneously.

Discord Message  
→ Observation Systems  
→ Social Graph Systems  
→ AI Systems  
→ Gameplay Systems  
→ Memory Systems  
→ Knowledge Systems  
→ Response Generation

This architecture allows the AI to behave as a **server intelligence engine** rather than a simple chatbot.
